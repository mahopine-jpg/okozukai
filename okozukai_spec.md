## 1. コンセプト

大人向けおこづかい管理アプリ。毎月の予算を用途別の「口座」に分けて管理し、日々の記帳で残高をリアルタイムに把握できる。週次・月次のAI評価で振り返りができ、欲しいものの購入相談や貯金計画まで一貫して管理できる。

**基本方針：**
- 操作は極力シンプルに
- インストール不要・アカウント不要でURLを開くだけで使える
- 記帳をできるだけ素早く、残高がひと目でわかるUI
- 固定費（ガソリン・タバコ）を差し引いた自由費を自分でコントロールする練習の場
- 月の予算は定額・残金は貯金口座に繰り越す
- 記帳履歴は削除せず永続蓄積する

**お金の流れ：**
- 代表口座（デビット用）に毎月¥37,500が入る
- 財布（現金）・PayPayへ「口座移動」しながら使う
- 月末に残金を貯金口座に積み立て、各口座をリセット

## 4. 画面構成・機能詳細

### 4-1. ホーム画面

- 今月の残高合計（大きく表示）
- 予算総額・使用済み金額のサマリ
- 残高割合のプログレスバー
- **🏦 貯金口座カード**（累計積立額を常に表示、一番上に固定）
- 口座カード一覧
  - 口座名・使用率（%）・残高・予算額
  - 使用率に応じてバーの色が変化（0〜69%：緑、70〜89%：オレンジ、90%以上：赤）
  - カードをタップすると記帳パネルが開き、該当口座が選択済みになる

### 4-2. 記帳パネル（画面下部・固定表示）

パネルハンドルをタップして開閉。「支出」「口座移動」の2タブ構成。

**支出タブ：**
- 口座セレクト
- レシート読み取りボタン（APIキー設定時のみ表示）
- クイック金額ボタン（¥500〜¥10,000）
- 金額入力欄・メモ欄・日付欄
- 「記帳する」ボタン（金額未入力時はグレーで押せない）
- 編集モード時はタイトルが「記帳を修正する」、ボタンが「修正を保存する」に変化

**口座移動タブ：**
- 移動元・移動先の口座セレクト（矢印で接続）
- クイック金額ボタン・金額入力欄・メモ欄・日付欄
- 「移動する」ボタン
- 実行すると `type:'transfer'` の2件（out/in）がペアで保存される

### 4-3. 履歴画面

- 全記帳を日付の新しい順に表示（月次繰越後も残る）
- 支出：口座タグ（グレー）・メモ・金額（赤）・日付・編集ボタン（✎）・削除ボタン（×）
- 口座移動：方向タグ（青）・相手口座名・メモ・金額（出：赤 / 入：緑）・日付
  - 移動の削除は出（out）側の × からペア2件まとめて削除

### 4-4. 集計画面（4タブ構成）

#### ① 週次タブ

- 週の区切りは**土曜〜金曜**
- 「‹ ›」ボタンで週を切り替え（今週より未来には進めない）
- ラベル例：「今週（4/19〜4/25）」「先週（4/12〜4/18）」
- 表示内容：今週の支出合計・今月の残高（残日数付き）・タバコ代今月累計・自由費残り
- 「AIに週次チェックしてもらう」ボタン → AIコメント表示
- 「📧 週次レポートをメールで共有」ボタン → `mailto:` でメールアプリ起動（本文自動生成）

**週次AIプロンプトの観点：**
- 今週の支出明細（メモ別）
- タバコ代の評価（目標より少なければ思いっきり褒める）
- 自由費の使い方・配分のバランス
- 来週の自由費をどう使うかのアドバイス
- 検討中の欲しいものリストも提示

**週次メール本文の内容：**
- 対象週（土〜金）・今週の支出・今月の残高・タバコ代
- 検討中の欲しいもの一覧（名前・価格・貯金計画）
- AIコメント全文

#### ② 購入相談タブ

- 欲しいもの名前・金額を入力
- 「AIに相談する」ボタン → AIアドバイス表示
- 「⭐ 欲しいものリストに追加」ボタン → 欲しいものリストに登録

**購入相談AIプロンプトの観点：**
- 今月残高・自由費残り・貯金口座残高を参照
- 今月すぐ買えるか判断
- 買えない場合：何をいくら節約すれば何ヶ月で買えるか（具体的数字）
- 貯金から出す場合の影響

#### ③ 欲しいものリストタブ

- 購入相談からリストに追加されたアイテムの一覧
- 状態：検討中 / 買った / 諦めた の3種類
- 「買った」にすると購入日（YYYY-MM-DD）が自動記録される
- 各アイテムの削除ボタン（確認ダイアログあり）
- 週次メール・月次AIに自動連携される

#### ④ 月次タブ

- 「‹ ›」ボタンで対象月を切り替え
- 「月次分析・AI評価を出す」ボタン
  - メモ別支出集計をバーグラフで表示（タバコ表記を自動正規化）
  - Claude APIによるAI評価コメント
  - 今月分の場合のみ「🏦 月次繰越する」ボタンが出現
- CSV出力ボタン（明細・全月サマリ）
- 月次サマリ履歴（過去の繰越済み月を一覧表示）

**月次AIプロンプトの観点：**
- メモ別支出内訳・予算・自由費実績
- 無駄遣いと思われる項目の具体的指摘
- タバコ代の評価（目標比較）
- 自由費の配分・月全体でのコントロール評価
- 今月購入した欲しいものも提示
- 来月の改善アドバイス

**月次繰越の動作：**
1. 各口座の残金合計を計算
2. 月次サマリ（口座別集計＋AI評価＋欲しいもの購入実績）を保存
3. 貯金口座に残金を加算
4. 今月の記帳に `rolledOver:true` フラグを付与（履歴は削除しない）
5. ホームの各口座残高が予算額に戻る
6. トーストで積立額・累計を通知

### 4-5. 設定画面

**口座と予算：**
- 口座名・月の予算額を編集、追加・削除可能
- 変更があるときのみ「口座設定を保存」ボタンが活性化
- 未保存で他タブに移動しようとすると確認ダイアログ

**固定費・連絡先設定：**
- ガソリン代（デフォルト¥8,500）
- タバコ代上限（デフォルト¥10,000）
- 週次レポートの送り先メールアドレス
- 変更があるときのみ「固定費・連絡先を保存」ボタンが活性化

**Claude APIキー：**
- 入力・保存（保存後はマスク表示）
- 変更があるときのみ「APIキーを保存」ボタンが活性化

**データ管理：**
- 「⚠ すべてのデータをリセット」：記帳履歴・貯金・月次サマリ・欲しいものリストを削除。口座設定・APIキー・固定費設定は保持
- 「🗑 完全リセット」：全データ（APIキー・口座設定含む）を削除してデフォルト状態に戻す
- 確認ダイアログ2回あり

**ヘルプ：**
- 「📖 使い方・運用マニュアル」リンク → `okozukai_manual.html` に遷移

## 5. データ構造

### 口座（wlt_wallets）
```json
[
  { "name": "代表口座(デビット用)", "budget": 37500 },
  { "name": "財布(現金)",           "budget": 0 },
  { "name": "PayPay",              "budget": 0 }
]
```

### 記帳履歴（wlt_history）
```json
[
  {
    "id": "一意ID（timestamp+random）",
    "walletName": "財布(現金)",
    "memo": "スーパー",
    "amount": 1240,
    "year": 2025,
    "month": 4,
    "day": 14,
    "ts": 1713234567890,
    "rolledOver": false
  },
  {
    "id": "xxx_out",
    "type": "transfer",
    "transferDir": "out",
    "transferPairId": "xxx",
    "transferPeerName": "PayPay",
    "walletName": "代表口座(デビット用)",
    "memo": "PayPayにチャージ",
    "amount": 3000,
    "year": 2025,
    "month": 4,
    "day": 14,
    "ts": 1713234567890
  },
  {
    "id": "xxx_in",
    "type": "transfer",
    "transferDir": "in",
    "transferPairId": "xxx",
    "transferPeerName": "代表口座(デビット用)",
    "walletName": "PayPay",
    "memo": "PayPayにチャージ",
    "amount": 3000,
    "year": 2025,
    "month": 4,
    "day": 14,
    "ts": 1713234567890
  }
]
```

**typeフィールドの仕様：**
- 通常支出：`type` フィールドなし（または undefined）
- 口座移動：`type: "transfer"`、`transferDir: "out"` または `"in"`
- `rolledOver: true` は月次繰越済みを示す。ホームの今月残高計算から除外されるが、履歴・過去月集計では参照される
- 残高計算で `transferDir: "in"` のレコードは支出から差し引いてプラスとして扱う

### 貯金残高（wlt_savings）
```json
46000
```

### 月次サマリ（wlt_summary）
```json
[
  {
    "year": 2025,
    "month": 4,
    "wallets": [
      { "name": "代表口座(デビット用)", "budget": 37500, "spent": 28400 }
    ],
    "totalRem": 9100,
    "aiComment": "今月は全体的によく管理できていました...",
    "ts": 1714567890123
  }
]
```

### 欲しいものリスト（wlt_wishlist）
```json
[
  {
    "id": "一意ID",
    "name": "AirPods",
    "price": 25000,
    "plan": "AIアドバイス全文テキスト",
    "status": "considering",
    "addedAt": "2025-04-14T10:00:00.000Z",
    "boughtAt": null
  }
]
```

**statusの値：** `"considering"` / `"bought"` / `"gave-up"`

### 固定費・連絡先（wlt_fixed）
```json
{
  "gas": 8500,
  "tobacco": 10000,
  "email": "parent@example.com"
}
```

### タバコ判定
メモ欄に「タバコ」「たばこ」「煙草」のいずれかが含まれる場合にタバコ代として集計。月次分析のメモ別集計では「タバコ」に正規化して表示。
```javascript
const isTobacco = memo => memo && /タバコ|たばこ|煙草/.test(memo);
```

## 7. 運用方法

### 毎日
1. ホームの口座カードをタップして記帳
2. メモ欄に品目を入力（タバコ購入時は「タバコ」「たばこ」「煙草」のいずれかを入力）
3. 口座移動（PayPayチャージ等）も記帳パネルの「口座移動」タブから記録

### 毎週金曜夜
1. 集計タブ →「週次」
2. 「AIに週次チェックしてもらう」を実行
3. 「📧 週次レポートをメールで共有」で親に送信

### 毎月末
1. 集計タブ →「月次」で「月次分析・AI評価を出す」を実行
2. 「🏦 月次繰越する」をタップ
3. 必要に応じて「📥 Excelに出力（CSV）」「📊 月次サマリCSVを出力」

### ファイルの更新方法（GitHub）
1. `github.com/mahopine-jpg/okozukai` を開く
2. 更新するファイルをクリック → 鉛筆アイコン（Edit）
3. `Ctrl+A` で全選択 → Delete → 新しい内容を貼り付け
4. 「Commit changes」をクリック（数分後に反映）

### APIキーの共有
- Claude APIキー（`sk-ant-api03-...`）をLINEなどで送り、設定画面で入力・保存
- 月の利用料は数十円以下

### デバイスとデータの関係
- localStorageはデバイス・ブラウザ単位で独立
- SafariとPWA（ホーム画面追加）は別のlocalStorage
- **必ずホーム画面のアイコン（PWA）から起動すること**
- ブラウザ側に残った不要なデータは設定 →「🗑 完全リセット」で消せる

## 11. 今後の課題

### 機種変更・マルチデバイス対応（GAS構成への移行）

**現状の課題：**
- 機種変更時にデータが引き継げない
- 複数デバイスで同じデータを見られない
- SafariとPWAでデータが分断される

**解決方針：GAS + Googleスプレッドシート構成**
```
スマホブラウザ → GAS Web App → Googleスプレッドシート
```

| 機能 | 現在 | 移行後 |
|------|------|--------|
| データ保存 | localStorage | Googleスプレッドシート |
| 認証 | なし | パスコード or Googleログイン |
| データ共有 | 不可 | どこからでも同じデータ |
| 機種変更 | データ消失 | データ保持 |

**必要な作業：**
1. スプレッドシートのシート設計（wallets・history・summary・savings・wishlist・fixed）
2. GAS Web Appの作成（GET/POSTエンドポイント）
3. フロントエンドのデータ保存処理をAPI呼び出しに置き換え
4. 簡易認証の実装（パスコード方式が最もシンプル）

※ おうちストックプロジェクトのGAS Web App実装パターンを流用できる。
